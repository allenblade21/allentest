# ADR 0004 — OCR 结果必须人工确认后才入账

状态:✅ 生效 · 2026-07

## 背景

视觉大模型识别虽好但非 100% 准确,金额/日期可能识别错或漏。若直接写入账本,错账会污染汇总且难追溯。

## 决策

OCR 识别结果一律先进入**「待确认清单」**(不落库),用户可逐条修改商户/分类/日期/金额;确认勾选后才批量写入正式 `transactions` 表(`source=ocr`,保留原图引用)。

- 低置信度标「存疑」、疑似重复标「疑似重复」,二者默认不勾选。
- 去重依据:金额 + 日期 + 商户 三项与已有账目匹配。

## 后果

- ✅ 识别错误不会静默污染账本;人机校对形成质量兜底。
- ✅ 用户修正商户→分类后,写入 `merchant_rules` 学习,越用越准。
- 约束:识别接口(`/api/ocr`)只返回待确认项,入账走独立接口(`/api/ocr/confirm`)。
