# ADR 0014 · 异常支出检测:近 3 月基线 + 固定倍数阈值

- 状态:已采纳(2026-07-19);P3 收官项
- 关联:`src/lib/anomaly.ts`、首页提醒条、分析页区块;TC-AX

## 决策

1. **基线 = 近 3 个自然月**(与周期候选识别同窗口):单笔异常 = 金额 > 3× 该分类基线单笔均值(基线需 ≥3 笔);分类异常 = 当月总额 > 2× 基线月均值。
2. **噪音抑制**:两类均要求金额 ≥ ¥50 才报;基线不足(笔数 <3 / 无支出)不报——宁漏勿扰。
3. **纯函数实时算、不落表、不可配置**:阈值写死在默认参数,个人场景不值得做设置界面;单笔与分类异常可同时报(大额单笔推高月总额属预期,信息互补)。
4. 呈现走页面架构的既定扩展位:首页 🔎 提醒条(点击进分析页)+ 分析页红色区块(最多列 5 项,标注基线值)。

## 备选与否决理由

- 标准差/Z-score:3 个月样本太小,统计意义弱且难解释;固定倍数用户能看懂"为什么报"。
- 机器学习/LLM 判断:成本与不可解释性远超收益;留给 P4 AI 问答做自然语言解读。

## 影响

- 零新表零依赖;TC-AX 4 例(单笔触发、正常不报、分类累积触发、噪音抑制)。
- P3 全部完成;首页提醒条现有三类:到期(琥珀)、超支(红 ⚠️)、异常(红 🔎)。
