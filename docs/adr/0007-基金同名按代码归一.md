# ADR 0007 — 基金按代码归一,同基金同日一条快照

状态:✅ 生效 · 2026-07(P1 已落地)

## 背景

进阶需求:同一基金多次导入持仓截图,要能汇总不同时期涨跌幅。同名基金可能是不同产品(A/C 类),不能只按名称合并;同一天重复导入也不应产生重复快照。

## 决策

- 基金主档 `funds` 以**基金代码为主键**;同名不同代码视为不同基金。OCR 只识别到名称、无代码时提示用户绑定代码。
- 每次录入是一条带日期的 `fund_records` 快照;在 `(fund_code, date)` 上加**唯一约束**——同基金同日只允许一条,重复导入提示覆盖或跳过。
- 跨期涨跌幅 = 相邻两期快照之间按市值计算。

## 后果

- ✅ 归一口径明确,跨期序列干净无重复。
- ✅ 唯一约束在 DB 层兜底,不依赖应用逻辑。
- 已落地:`lib/fund-db.upsertFundRecord` 按代码归一 + 同日 upsert 覆盖;详情页跨期表 + 走势图。
