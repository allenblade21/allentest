# ADR 0002 — 金额用整数「分」存储

状态:✅ 生效 · 2026-07

## 背景

记账涉及大量金额加减与汇总。浮点数(如 0.1 + 0.2)会产生精度误差,累加后结余对不上,对账本是致命的。

## 决策

所有金额一律以**整数「分」**(cents)存储于数据库与内存计算;仅在展示层转换为「元」字符串。转换集中在 `src/lib/money.ts`(`formatCents` / `parseYuanToCents`)。

## 后果

- ✅ 加减汇总全是整数运算,零精度误差。
- ✅ 展示/解析逻辑单点收敛,易测。
- 约束:任何新字段涉及金额都必须存分(如 `amountCents`、`marketValueCents`、`holdingProfitCents`),API 出入参也用分。
