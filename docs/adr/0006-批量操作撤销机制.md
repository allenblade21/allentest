# ADR 0006 — 批量操作用「操作前快照 + 还原」实现撤销

状态:✅ 生效 · 2026-07

## 背景

批量改分类/账户/日期/删除影响多条记录,误操作(尤其批量删除)代价大,需要可靠的「一次撤销」。

## 决策

批量接口 `/api/transactions/batch` 在执行前**取出被影响行的完整快照**并随响应返回;前端 8 秒内可点「撤销」,调 `/api/transactions/restore` 用快照**按原 id 整体还原**(删除的重新插入、改过的覆盖回旧值)。

## 后果

- ✅ 撤销恢复到操作前的精确状态,含被删记录的原 id。
- ✅ 通用于四种批量操作,无需为每种单独设计逆操作。
- 教训:撤销 toast 是全局 UI,不能只在「有数据」分支渲染——批量删空后仍需能撤销(见 [memory/lessons.md](../memory/lessons.md),由 TC-B3 发现)。
