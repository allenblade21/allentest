# ADR 0001 — 技术栈:Next.js 全栈 + SQLite + LLM 视觉 OCR

状态:✅ 生效 · 2026-07

## 背景

个人自用记账软件,手机优先 Web,OCR 是核心,数据要自己掌控。需在开发效率、OCR 质量、成本、多设备同步之间权衡。评估了三个方案:A(Next.js 全栈+LLM 视觉)、B(Vue+FastAPI+本地 PaddleOCR)、C(纯前端 PWA)。

## 决策

选方案 A:Next.js(App Router)+ TypeScript + Tailwind + SQLite/Drizzle + 视觉大模型 OCR。详见 [`docs/技术选型.md`](../技术选型.md)。

## 后果

- ✅ 前后端一体,开发最快;LLM 直接出结构化 JSON,省去传统 OCR 的解析规则维护。
- ✅ SQLite 单文件,备份=拷文件;天然支持多设备(服务端走 DB)。
- ⚠️ OCR 按量付费(个人量级几元/月);截图需经 API(用人工确认兜底)。
- 被否:B(解析规则维护成本高)、C(数据锁在单浏览器,不能多设备同步)。
