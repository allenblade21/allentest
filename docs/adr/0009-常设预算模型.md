# ADR 0009 — 预算用「常设月度额度」而非按月记录

状态:✅ 生效 · 2026-07

## 背景

路线图草案里 `budgets` 表带「生效月」字段(每月单独设预算)。但对个人自用场景,逐月设置预算是负担——绝大多数人希望"餐饮 500/月"设一次、月月生效。

## 决策

`budgets` 表为**常设额度**:`(categoryId 唯一, limitCents)`,不带月份字段;额度对每个月生效。当月执行进度 = 该分类当月支出 / 额度(`lib/budget.budgetProgress`,复用 analytics 的分类聚合)。保存接口语义:`limitCents > 0` upsert,`= 0` 删除。

## 后果

- ✅ 设置一次月月适用,UX 最简;表结构与接口都极小。
- ⚠️ 不支持"某月临时调预算"——改额度会影响所有月份的回看口径。个人使用可接受;若未来需要,再加按月覆盖表,不破坏现有结构。
- 展示约定:>100% 红色+「超支」标签,≥80% 琥珀色预警,首页只在实际超支时出提醒卡。
