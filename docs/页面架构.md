# 页面架构设计 — 拆分原则与扩展位规划

> 目标:每个页面职责单一、区块可插拔,新功能"插区块/加入口",不重排旧页面。
> 配套演进:P3 异常检测、P4 PWA/通知/AI 问答/自动记账(见 [路线图](路线图.md))。

## 一、设计原则

1. **三击可达**:任何高频操作 ≤3 次点击(记一笔=1 击,导入=2 击,其余经「我的」≤3 击)。
2. **首页只做两件事**:看结果(hero 汇总 + 流水)、接收提醒(单行提醒条)。所有管理/设置一律下沉二级页。
3. **区块式页面**:页面 = 区块(卡片)自上而下堆叠,区块间零耦合;新功能优先作为**新区块**或**新提醒条**插入,而非改造既有区块。
4. **入口注册制**:「我的」页由 `groups` 数据驱动,新页面 = 配置加一行;首页提醒条按「提醒源」聚合,新提醒 = 新增一个提醒源函数。
5. **计算归 lib、页面归展示**:所有聚合/判定是 `lib/*` 纯函数(可单测),server 页面只做「查库 → 调纯函数 → 渲染区块」。

## 二、页面拆分(现状 → 组件化 → 扩展位)

### 1. 首页 `/`(流水)
| 区块 | 拆分为 | 未来扩展 |
|------|--------|----------|
| hero 汇总 | `HomeHero`(月份切换/导入/结余/收支) | 支出预测数字(P4 自动记账) |
| 提醒条 | `AlertStrip`(通用单行条)× 提醒源列表 | **异常支出提醒即新提醒源**(P3 迭代 4);账单/通知类提醒复用 |
| 流水列表 | `TransactionList`(已有,含批量模式) | 搜索/筛选工具条(可选增强) |

> 提醒源约定:`lib/alerts.ts` 聚合 `{type, severity, text, href}[]`,首页只 map 渲染 `AlertStrip`,新增提醒零改首页。

### 2. 记一笔 `/record`
拆分:`TypeSegment`(支出/收入/转账)、`AmountDisplay + Keypad`、`CategoryGrid`、`MetaFields`(账户/日期/备注)。
扩展位:键盘上方预留「智能建议行」——P4 自动记账在此显示"周期支出待确认"或商户猜测,不挤压主流程。

### 3. OCR 导入 `/import`、基金导入 `/funds/import`
拆分:`Uploader`(选图/预览)、`PendingList`(待确认清单,行组件交易/基金各自实现,**勾选-存疑-入账交互统一**)。
扩展位:清单底部"识别为周期支出?"建议区(与 TC-RC 候选识别打通)。

### 4. 基金 `/funds`、详情 `/funds/[code]`
拆分:`FundOverviewCard`(总市值/持有收益)、`FundList`;详情 `SnapshotTable` + `TrendChart`。
扩展位:总览卡加"占净资产比例"(与 /networth 联动);详情加基准对比线(远期)。

### 5. 消费分析 `/analysis`
拆分:`MomCard`(环比)、`TrendBars`(6 月趋势)、`CategoryBars`(分类结构)、`BudgetProgressList`(预算执行)。
扩展位:**异常检测区块**(P3 迭代 4:偏离历史均值的分类/单笔标注)插在环比卡之后——分析页是异常明细的天然承载页,首页只挂提醒条。

### 6. 单一职责页(已合理,不再拆)
`/budget`(预算设置)、`/recurring`(周期支出)、`/networth`(净资产)、`/report`(月度报告)、`/login` `/register`。
- `/report` 扩展位:底部"AI 总结一句话"区块(P4 AI 问答的最小落点);导出 PDF 按钮。
- `/networth` 扩展位:趋势小图(按月快照净资产,需再议是否落表)。

### 7. 我的 `/me`
现状已是 `groups` 配置驱动 ✅。规划分组:**分析与预算**(现有 5 入口)/ **数据管理**(分类、账户、导出 CSV——待实现)/ **设置**(通知开关、OCR 配置状态、关于)/ 退出登录。
新页面上线 = groups 加一行,同时在 `PUBLIC_PATHS` 之外自动受登录保护(铁律 9)。

## 三、共享 UI 组件(消重复)

现状大量重复 className,抽 `src/components/ui/`:
`Card`(细描边卡)、`AlertStrip`、`PageHeader`(‹ 标题 + 右侧动作)、`ProgressBar`(含超支红态)、`Money`(tabular-nums 金额)、`Segment`。
**约束**:抽取是纯重构,行为不变,以现有 55 E2E 作保障;一次抽一类,跑绿再下一类。

## 四、落地顺序

1. 抽共享 UI 组件 + `lib/alerts.ts` 提醒源聚合(纯重构,测试保障)
2. P3 迭代 4 异常检测:按本设计以"提醒源 + 分析页区块"落地(TC-AX)
3. 「我的 → 数据管理」组:导出 CSV → 分类/账户管理
4. P4 各项按扩展位插入,不再动页面骨架
