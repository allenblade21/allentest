# 测试用例登记表

> 端到端测试基于 Playwright,手机视口(390×844)+ 独立测试库(`data/test.db`)+ OCR mock,不触碰开发数据。
> 用例编号与 `tests/*.spec.ts` 中的测试标题一一对应,新增用例时先在此登记再写代码。

## 入口(如何跑 / 如何看)

| 命令 | 作用 |
|------|------|
| `npm test` | 完整跑:先构建再执行全部用例 |
| `npm run test:quick` | 快速跑:复用已有构建(改了测试没改代码时用) |
| `npx playwright test tests/ocr.spec.ts` | 只跑某个套件 |
| `npx playwright test -g "TC-T5"` | 只跑某个用例(按编号过滤) |
| `npm run test:list` | 列出全部用例(不执行) |
| `npm run test:report` | 打开上次运行的 HTML 报告(含每个用例的页面截图、失败 trace) |
| `npm run test:doc` | 由最近一次运行结果生成 [测试报告.md](测试报告.md)(按功能模块汇总,入库存档) |

报告产物:`playwright-report/`(HTML + `results.json`,已 gitignore)。每个用例自动截图,失败时保留 trace 可回放。
