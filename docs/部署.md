# 部署指南 — macOS 家用服务器(MacBook 常开)

> 已定部署形态(ADR 0011):**家里常开的 MacBook + Tailscale 内网访问**。
> 应用已内置登录/注册(ADR 0012):首次访问先注册账号;自己(和家人)注册完后,
> 在 `.env.local` 加 `ALLOW_REGISTER=0` 关闭注册并重启服务。
> 仍推荐 Tailscale 内网、不做公网端口映射(纵深防御);真要公网还需 HTTPS 反代。

## 总览

```
手机(Tailscale)──┐
                    ├── Tailscale 加密内网 ──► MacBook(launchd 常驻 npm start :3000)
平板/其他电脑 ─────┘                              └── data/app.db + data/uploads(每日 03:30 备份到 backups/)
```

## 一、首次部署(MacBook 终端执行)

```bash
git clone https://github.com/allenblade21/allentest.git
cd allentest
npm run setup            # 装依赖 + 建库 + 默认数据 + 生成 .env.local
open -t .env.local       # 填 ARK_API_KEY(或 OCR_MOCK=1 先体验)
npm run build
bash scripts/macos-setup-server.sh   # 常驻自启 + 每日备份,一键配置
```

首次打开 `http://localhost:3000` 会跳到登录页 → 点「去注册」创建账号。
家人也要用的话让他们各自注册;注册完在 `.env.local` 加一行 `ALLOW_REGISTER=0`,
然后 `launchctl kickstart -k gui/$(id -u)/com.jizhangben.app` 重启生效。

脚本做了什么:

| 项 | 内容 |
|----|------|
| 应用常驻 | launchd `com.jizhangben.app`:开机自启、崩溃自动拉起,监听 `0.0.0.0:3000` |
| 自动备份 | launchd `com.jizhangben.backup`:每日 03:30 跑 `npm run backup` |
| 日志 | `data/logs/app.log` / `app.err.log` / `backup.log` |

## 二、防休眠(必做)

- 插上电源,执行:`sudo pmset -c sleep 0`(仅插电时禁止系统休眠,不影响用电池时)
- **合盖会断网断服务**:建议开盖放置;必须合盖用则外接显示器,或装 Amphetamine 一类工具
- 「系统设置 → 电池」确认插电时不自动进入睡眠

## 三、手机随时访问(Tailscale)

> 安卓手机的逐步操作(含无 Google Play 的情况、加主屏、排查)见 [部署-安卓端.md](部署-安卓端.md)。

1. MacBook 和手机都安装 [Tailscale](https://tailscale.com)(App Store 有),登录**同一账号**
2. 手机浏览器访问 `http://<MacBook 的 Tailscale 机器名>:3000`(在 Tailscale App 里能看到名称/IP)
3. 加到主屏当 App 用:Safari 分享 → 「添加到主屏幕」(PWA 离线壳是 P4 后续项)

不装 Tailscale 的话,同一 Wi-Fi 下用 `http://<MacBook局域网IP>:3000` 也行,只是出门就断。
**仍不建议在路由器上做端口映射暴露公网**——虽已有登录保护(ADR 0012),但公网还需 HTTPS 反代,Tailscale 内网是更稳妥的默认。

## 四、备份与恢复

- 自动:每日 03:30 → `backups/<日期_时间>/`(SQLite 在线备份,不停服;含 OCR 原图;自动保留最近 30 份)
- 手动:`npm run backup`
- 异地容灾:Time Machine 会顺带覆盖 `backups/`;更稳可加 rclone 把 `backups/` 同步到 iCloud/网盘
- **恢复**:停服务 → 用备份里的 `app.db` 覆盖 `data/app.db`(同时删掉 `data/app.db-wal` / `-shm`)→ 重启服务

```bash
launchctl unload ~/Library/LaunchAgents/com.jizhangben.app.plist
cp backups/<某份>/app.db data/app.db && rm -f data/app.db-wal data/app.db-shm
launchctl load -w ~/Library/LaunchAgents/com.jizhangben.app.plist
```

## 五、更新版本

```bash
cd allentest
git pull
npm run setup && npm run build
launchctl kickstart -k gui/$(id -u)/com.jizhangben.app   # 重启服务
```

## 六、故障排查

| 现象 | 查什么 |
|------|--------|
| 打不开页面 | `launchctl list \| grep jizhangben` 看进程在不在;`tail -50 data/logs/app.err.log` |
| 手机连不上 | 两端 Tailscale 是否都在线、同一账号;MacBook 是否休眠了 |
| OCR 识别失败 | `.env.local` 的 `ARK_API_KEY` 是否已填、ModelArk 控制台是否开通了 Seed-2.0-lite |
| 备份没生成 | `tail data/logs/backup.err.log`;03:30 时机器必须是醒着的(见防休眠) |
