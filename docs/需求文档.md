## 1. 产品概述

### 1.1 定位

一款**个人使用**的自动化记账工具。核心卖点是 **OCR 批量导入**:把支付宝/微信账单截图、纸质小票、基金 App 截图直接变成账目记录,把手动录入的摩擦降到最低。同时提供基金投资的跨期收益追踪。

### 1.2 设计原则

1. **录入摩擦最小化**:能 OCR 就不手打,能批量就不逐笔。
2. **简洁易用**:核心操作(记一笔、看流水)不超过 2 次点击可达;界面只保留必要元素。
3. **人机校对兜底**:所有 OCR 识别结果必须经用户确认后才正式入账,避免错账。
4. **数据归属个人**:数据本地存储或自托管,支持导出,不依赖第三方账号。

### 1.3 功能优先级总览

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 基础记账(手动 + OCR) | 本期核心交付 |
| P1 | 基金记账与投资管理 | 本期进阶交付 |
| P2 | 推荐功能(分析/预算/提醒) | 占位,仅记录需求,以后再做 |

## 2. 基础功能 — 记账模块(P0)

### 2.1 手动记一笔

- 字段:**金额**(必填)、**类型**(支出/收入/转账)、**分类**(必填,默认支出分类)、**账户**(现金/微信/支付宝/银行卡等)、**日期**(默认今天)、**备注**(选填)。
- 交互:进入页面即弹出金额数字键盘;分类以宫格图标呈现,常用分类排前;保存后自动返回流水页。
- 支持「保存并再记一笔」,连续录入不返回。

### 2.2 编辑已有条目

- 从流水列表点击任意条目进入详情/编辑页,可修改全部字段。
- 支持删除(带确认提示);OCR 导入的条目保留原始截图缩略图供比对。

### 2.3 批量修改

- 流水列表长按(或点「编辑」按钮)进入多选模式。
- 多选后可执行:**批量改分类**、**批量改账户**、**批量改日期**、**批量删除**。
- 顶部提供「全选本页 / 按筛选条件全选」。
- 批量操作前显示影响条数,操作后支持一次撤销(Undo)。

### 2.4 OCR 单张录入

- 入口:记账页的「拍照/相册」按钮。
- 流程:拍照或选择 1 张图片 → 识别 → 生成 1 条**待确认记录**(金额、商户→备注、时间、推测分类)→ 用户校对修改 → 保存入账。
- 支持来源:纸质小票/发票拍照、单笔支付详情截图。

### 2.5 OCR 批量录入

- 入口:首页「导入」按钮。
- 流程:一次选择多张图片(支付宝/微信账单列表截图、银行 App 交易明细截图、电子对账单)→ 逐张识别 → 汇总为**待确认清单**。
- 待确认清单能力:
  - 每笔显示识别出的金额/商户/时间/推测分类,可逐笔点开修正;
  - 识别失败或存疑的条目标黄置顶,提示人工处理;
  - **自动去重**:与已有账目「金额 + 时间 + 商户」三项匹配的条目标记为疑似重复,默认不勾选;
  - 「全选入账」一键批量保存,或仅勾选部分入账。
- 分类推测:根据商户关键词自动映射分类(如「美团」→餐饮),用户修正后记住映射,越用越准。

### 2.6 流水列表(首页)

- 顶部**月度汇总卡**:本月支出、收入、结余;左右滑动/点击切换月份。
- 流水**按日分组**,每组显示当日小计;条目显示分类图标、备注/商户、账户、金额。
- **搜索与筛选**:关键词搜索备注/商户;按分类、账户、类型、时间段筛选。
- 悬浮「+」按钮直达记一笔;「导入」按钮直达 OCR 批量录入。

### 2.7 分类与账户管理

- 内置常用分类(餐饮、交通、购物、居住、娱乐、医疗、工资、理财…),支持增删改、排序、自定义图标颜色。
- 账户管理:增删改账户,账户可设初始余额,流水自动更新余额。

## 3. 进阶功能 — 基金记账与投资管理(P1)

### 3.1 基金 OCR 批量录入

- 入口:基金页「导入持仓截图」。
- 支持来源:支付宝基金、天天基金、银行/券商 App 的**持仓列表截图**和**单只基金收益截图**。
- 识别字段:基金名称、基金代码(无代码时按名称模糊匹配主档)、持有金额/份额、当日涨跌幅、持有收益、截图对应日期(识别不到则默认今天,用户可改)。
- 一次可传多张截图,识别结果进入待确认清单(交互同 2.5),确认后生成**基金快照记录**。

### 3.2 同名基金跨期汇总

- 同一基金按**基金代码归一**(同名不同代码视为不同基金;OCR 只识别到名称时提示用户绑定代码)。
- 每次导入/手动录入形成一条带日期的快照,同一基金的快照按时间排列为**时间序列**:
  - 表格视图:各期日期、市值、当日涨跌幅、区间涨跌幅(相邻两期之间)、持有收益;
  - 图表视图:市值/收益走势折线图;
  - 汇总:该基金累计收益、持有期间总涨跌幅、最高/最低点。
- 同一天重复导入同一基金:提示覆盖或跳过,不产生重复快照。

### 3.3 投资总览

- 基金页顶部**总览卡**:总市值、总持有收益、今日收益(基于各基金最新快照汇总)。
- 基金列表:每只基金显示最新市值、持有收益、最近涨跌幅,点击进入 3.2 的详情页。
- 支持手动记一笔基金快照(不走 OCR),以及对错误快照的修改/删除。

## 4. 推荐功能 — 占位(P2,本期不设计不实现)

> 以下仅记录需求方向,细化设计留待后续版本。

- **消费分析**:按分类/月份的支出结构图,同比/环比变化。
- **预算建议**:按历史支出自动建议各分类月度预算,超支提醒。
- **异常支出提醒**:偏离个人消费习惯的大额/高频支出提示。
- **订阅与固定支出识别**:自动识别周期性扣费(会员、房租),到期前提醒。
- **月度报告**:自动生成本月收支+投资摘要。

## 5. 非功能需求

| 项 | 要求 |
|----|------|
| 响应式 | 手机优先;桌面浏览器可用(列表加宽、批量操作更高效) |
| 性能 | 万级流水条目下列表滚动与筛选流畅;OCR 单张识别 < 10s |
| OCR 准确性 | 识别仅产生「待确认记录」,入账必须经人工确认;失败可回退手动录入 |
| 数据安全 | 单用户,本地/自托管部署;数据支持一键导出 CSV(流水、基金快照分别导出) |
| 图片存储 | OCR 原图保留(可设置保留期限),供事后比对 |

## 6. 数据模型草案

| 实体 | 关键字段 | 说明 |
|------|----------|------|
| `Transaction` 流水 | id, 类型(支出/收入/转账), 金额, 分类 id, 账户 id, 日期, 备注/商户, 来源(手动/OCR), 原图引用 | 核心账目记录 |
| `Category` 分类 | id, 名称, 图标, 颜色, 类型, 排序 | 支持自定义 |
| `Account` 账户 | id, 名称, 类型, 初始余额 | 余额由流水推算 |
| `Fund` 基金主档 | 代码(主键), 名称 | 同名基金按代码归一 |
| `FundRecord` 基金快照 | id, 基金代码, 日期, 市值/份额, 当日涨跌幅, 持有收益, 来源, 原图引用 | 同基金同日期唯一 |
| `OcrImportBatch` 导入批次 | id, 时间, 图片列表, 待确认项(字段+置信度+去重标记+状态) | 批量导入的暂存区 |
| `MerchantRule` 商户映射 | 商户关键词 → 分类 id | 分类推测的学习记录 |
