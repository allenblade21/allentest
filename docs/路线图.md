## P3 — 主动理财与资产全景

**主题**:从「被动记录」到「主动提醒」,并让个人财务数据可信、可视为一个整体。全部建立在现有 `transactions` / `funds` 之上,仅新增 `budgets`、`recurring` 两张表,风险低。

| 功能组 | 内容 | 数据模型 | 新页面 / API |
|--------|------|----------|-------------|
| 预算体系 | 分类月度预算,分析页/首页显示进度与超支 | 新表 `budgets`(categoryId, limitCents, 生效月) | `/budget`;分析页超支条 |
| 周期支出 | 订阅/房租识别与登记,到期提醒 | 新表 `recurring`(name, amountCents, 周期, 下次日期, categoryId) | `/recurring`;首页到期提示 |
| 异常检测 | 偏离习惯的大额/高频支出提示 | 无(基于历史统计) | 分析页「异常」区 |
| 月度报告 | 自动生成当月收支+投资摘要,可导出 | 无(聚合现有数据) | `/report/[month]` |
| 净资产总览 | 现金账户余额 + 基金最新市值合并为总资产 | 无(合并账户 + 基金快照) | 首页/我的「净资产」卡 |
| 数据备份 | 完整导出 JSON、导入恢复(财务数据不能丢) | 无 | `/api/backup` 导出/导入 |

**迭代拆分(建议顺序)**:
1. 预算体系 + 超支提示 — 测试组 TC-BG
2. 周期支出识别 + 到期提醒 — TC-RC
3. 净资产总览 + 月度报告 — TC-NW / TC-RP
4. 异常检测 + 数据备份恢复 — TC-AX / TC-BK

## P4 — 平台化与智能增强

**主题**:从「本地能跑」到「随处可用、更聪明」,让 App 真正日常可用。

| 功能组 | 内容 | 关键点 | 风险 / 依赖 |
|--------|------|--------|------------|
| 部署上线 | ✅ 形态已定并落地:MacBook 家用服务器 + Tailscale(ADR 0011,[部署.md](部署.md),launchd 常驻 + 每日备份);剩 PWA(加主屏、离线壳) | 不做 Docker(macOS 原生 launchd 更轻) | — |
| 访问保护 | 单用户口令登录(公网自托管必需) | 轻量 session,不做多用户 | 仅当将来公网暴露才需要(Tailscale 内网下不做) |
| 通知推送 | 预算超支 / 账单到期 / 异常 → PWA 推送或邮件 | 复用 P3 预算/周期数据 | 依赖 P3 |
| AI 财务问答 | 基于自己数据的对话(「上月餐饮花了多少」) | 复用现有 OCR provider 的 LLM 能力 | 近零新基建,可早做 |
| 自动记账/预测 | 周期支出自动生成待确认、下月支出预测 | 复用 recurring + 历史 | 依赖 P3 |

**迭代拆分**:
1. Docker 化 + PWA + 部署文档
2. 访问口令保护
3. 通知推送(超支/到期)
4. AI 财务问答
5. 自动记账 + 消费预测

## 依赖与建议节奏

```
P2剩余 ─┐
        ├─► P3 主动理财(预算→周期→净资产/报告→异常/备份)
        │        └─► P4 通知 / 自动记账(依赖 P3 数据)
        ├─► P4 部署 / PWA / 口令(不碰数据层,可与 P3 并行)
        └─► P4 AI 问答(依赖现有数据即可,可早插入)
```

- 先做完 **P3**(数据层增强、闭环价值高),再进 P4 平台化。
- **P4 部署/PWA** 不碰数据层,可随时并行插队。
- **P4 AI 问答** 几乎零新基建、体验亮眼,可作为激励性早期插入项。

## 每迭代流程(与前 10 迭代一致)

改代码 → 补测试(新 TC 组)→ 更新文档(STATUS / CHANGELOG / journal / 需要时 ADR)→ commit 推送 main。详见 [AGENTS.md](../AGENTS.md) 标准工作流。
